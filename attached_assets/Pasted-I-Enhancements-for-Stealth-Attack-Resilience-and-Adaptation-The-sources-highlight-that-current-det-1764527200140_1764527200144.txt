Vulnerability-Based Observability Configuration Tool
Allow users to define the Partial Observability configuration by weighting specific electrical and topological metrics using the Analytical Hierarchical Process (AHP). This includes metrics like effective graph resistance, load shedding, connectivity impact, and geodesic vulnerability, allowing utility operators to configure monitoring based on their specific critical infrastructure definitions.
The critical nodes are selected based on their overall vulnerability scores, derived using AHP to weigh electrical metrics (effective graph resistance, load shedding, etc.) and topological metrics (connectivity impact, geodesic vulnerability, etc.).
